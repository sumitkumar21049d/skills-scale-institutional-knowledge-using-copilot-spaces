# OctoAcme — Personas: Additions & Interaction Patterns

Purpose: Provide concise role descriptions and a one-line "when to involve" guideline for each additional persona to improve clarity during planning and execution.

- Delivery Lead
  - When to involve: During sprint planning, backlog grooming, and daily standups; whenever cross-team coordination is required.
- Technical Lead / Architect
  - When to involve: At design/architecture decisions, large PR reviews, and when technical trade-offs affect scope or schedule.
- Product Designer (UX)
  - When to involve: Early in planning and before final acceptance criteria are locked.
- Data Analyst / Insights Partner
  - When to involve: Before launch to define metrics and after release for analysis and iteration.
- Security / Compliance Reviewer
  - When to involve: During planning for sensitive features, PR review for security-critical changes, and before production releases.
- Release Coordinator
  - When to involve: When scheduling releases, producing release notes, and coordinating stakeholder communications.

Suggested usage:
- Add the "When to involve" line to each backlog item or PR template so reviewers and owners know when to call these roles in.
- Add a single-line owner field in the project board card templates indicating the persona who should be looped in for that item.
