# OctoAcme — Process Checklists (New templates)

This file contains checklists that make role responsibilities actionable and improve handoffs.

## Role Handoff Checklist (use when moving from planning → implementation)
- [ ] PdM has defined objective, metrics, and acceptance criteria.
- [ ] Delivery Lead confirmed sprint scope and capacity.
- [ ] Technical Lead reviewed and signed off on architecture/design or flagged TBD items.
- [ ] Product Designer provided final UX assets and acceptance criteria.
- [ ] Data Analyst included measurement plan or instrumentation tickets.
- [ ] Security Reviewer performed initial assessment (if applicable).

## Release Readiness Checklist (use before tagging a release)
- [ ] All acceptance criteria met and validated by QA.
- [ ] CI passing and security scans completed.
- [ ] Release notes drafted and reviewed by PdM.
- [ ] Rollback/mitigation plan documented and tested.
- [ ] Release Coordinator scheduled the window and notified stakeholders.
- [ ] Post-release verification plan assigned.
